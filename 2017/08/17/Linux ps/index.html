Linux PS Command

Description

ps(progress status) is a native Unix/Linux utility for viewing infomation concering a selection of runnig processes on a system: it reads this information from the virtual files in /proc_filesystem. It is one of the important utilities for system administration specifically under process monitoring, to help you understand what is going on a Linux system.

It has numerous options for manipulating its output, however you’ll find a small number of them practically useful for daily usage.

Useful Daily Command

  1. ps aux | grep nginx

To list processes with nginx.

  1. ps -v pid
  PID TTY      STAT   TIME  MAJFL   TRS   DRS   RSS %MEM COMMAND
 1180 ?        Ss     0:00      0  1079 124020 1508  0.0 nginx: master process /usr/sbin/nginx -g daemon on; master_process on;

Display virtual memory format. Check memory usage on the progress of pid.

  1. ps -p 1154 -o comm=

To find a process name using its PID, it’s useful for bash script.

  1. ps -eo pid,%cpu,%mem,cmd --sort=-%mem | less

Find top running processes by highest memory and CPU usage in Linux.

Important

ps only shows static information, to view frequently updated output you can use tools such as top.
