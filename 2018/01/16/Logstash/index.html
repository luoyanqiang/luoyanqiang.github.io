Logstash

Logstash is an open source, server-side data processing pipeline that ingests data from a multitude of sources simultaneously, transforms it, and then sends it to your favorite “stash”.
Logstahs是一个开源，服务端的数据处理管道，它同时从多源头摄取数据，转换数据并且随之传输到你喜欢的收藏地方。

Inputs generate events, filters modify them, and outputs ship them elsewhere. Description as follows.

INPUTS

Logstash supports a variety of inputs that pull in events from a multitude of common sources, all at the same time. Easily ingest from your logs, metrics, web applications, data stores, and various AWS services, all in continous, streaming fashion.
Logstash支持各种输入，这引起输入同时从众多公共源中提供事件。轻易地从你的日志、指标、web应用程序、数据存储，及其他各种AWS服务中摄取事件，所有都以连续、流方式进行。
Some of the more commonly-used inputs are: file, syslog, redis, beats

FILTERS

Parse & transform your data on the fly. As data travels from source to store, Logstash filters parse each event, identify named fields to build structure, and transform them to converge on a common format for easier, accelerated analysis and business value.
即时解释且转化你的数据。在数据从源头到存储的过程，Logstash过滤器会解释每个事件，识别被命名的字段来建立结构，转化它们汇聚成一个更容易分析商业价值的共同格式。
Some useful filters include: grok, mutate, mutate, drop, clone, geoip

OUTPUTS

Logstash has a variety of outputs that let you route data where you want, giving you the flexibility to unlock a slew of downstream use cases.
Logstash有多种输出，能够让你把数据路由到你希望的地方，让你更灵活解锁大量下游用例。
Some commonly used outputs include: elasticsearch, file, graphite, statsd
